# دورة حجز الشاليه (Booking Cycle)

توثيق كامل لدورة الحجز في تطبيق Sununo، يغطي نوعي الشاليه، الحالات، عملية الدفع،
ومنطق إظهار/إخفاء عناصر الواجهة حسب الحالة.

> الملفات المرجعية:
> - الباك إند: `sununo-api/src/bookings/services/bookings.service.ts`
> - إنشاء الحجز (العميل): `Sununo/app/(customer)/booking/complete.tsx`
> - تفاصيل الحجز (العميل): `Sununo/app/(tabs)/(customer)/booking-success.tsx`
> - تفاصيل الحجز (المالك): `Sununo/app/(dashboard)/booking-details.tsx`

---

## 1. نوع الشاليه (`chalet.bookingType`)

أول ما يُحدَّد في الدورة هو نوع الشاليه، وهو يحدد ما إذا كان الحجز يحتاج موافقة قبل الدفع:

| النوع | المعنى | متى يُدفع؟ |
|------|--------|-----------|
| `instant` | حجز فوري — لا يحتاج موافقة المالك | **يُدفع فوراً** عند إنشاء الحجز |
| `delayed` | حجز بانتظار الموافقة | **يُدفع بعد موافقة المالك**، بنفس طريقة الشاليه الفوري |

نوع الدفع (عربون أو كامل) يعتمد على `chalet.depositPercentage`:
- إذا كانت `depositPercentage > 0` → يستطيع العميل اختيار **عربون** أو **مبلغ كامل**.
- إذا كانت `depositPercentage = 0` → **مبلغ كامل فقط**.

---

## 2. حالات الحجز (`BookingStatus`)

```
pending_approval → pending_payment → confirmed → completed
                                   ↘ cancelled (إلغاء / رفض / انتهاء مهلة)
```

| الحالة | المعنى |
|--------|--------|
| `pending_approval` | بانتظار موافقة المالك (فقط للشاليه المؤجل `delayed`) |
| `pending_payment` | بانتظار دفع العميل |
| `confirmed` | تم الدفع وتأكيد الحجز (الإقامة لم تنتهِ بعد) |
| `completed` | انتهت فترة الإقامة / أكمل المالك الحجز |
| `cancelled` | مُلغى (إلغاء، رفض المالك، أو انتهاء مهلة الدفع) |

> لا توجد حالة `rejected` منفصلة — رفض المالك يُسجَّل كـ `cancelled` مع `cancellationReason`.

---

## 3. الدورة الكاملة

### أ) الشاليه الفوري (`instant`)

1. العميل يختار التاريخ/الفترة في `complete.tsx`.
2. يظهر له اختيار الدفع: **عربون** (إن وُجد) أو **كامل** + طريقة الدفع.
3. عند الإرسال → `createBooking` يُنشئ الحجز بحالة `pending_payment` ويبدأ الدفع مباشرة:
   - **محفظة** → خصم فوري → `confirmed`.
   - **بطاقة (wayl)** → فتح رابط الدفع → عند نجاح الدفع (webhook) → `confirmed`.
4. **إذا دُفع عربون → ينتهي الموضوع ويتم تأكيد الحجز** (`confirmed`)، ويبقى المبلغ المتبقي
   يُدفع عند الوصول (نقداً)، ويؤكده المالك لاحقاً.
5. تبقى حالة الحجز ظاهرة حتى **يُكمل المالك الحجز** أو تنتهي فترة الإقامة → `completed`.

### ب) الشاليه المؤجل (`delayed`)

1. العميل يختار التاريخ/الفترة. **لا تظهر خيارات الدفع** — بدلاً منها ملاحظة توضح أن الطلب
   سيُرسل للمالك.
2. عند الإرسال → `createBooking` يُنشئ الحجز بحالة `pending_approval` (بدون دفع).
3. المالك يوافق (`approveBooking`) → الحالة تصبح `pending_payment`.
   - **لا يُفرض الدفع الكامل** — نوع الدفع (عربون/كامل) يُحسم وقت دفع العميل، تماماً كالشاليه الفوري.
4. العميل يفتح **تفاصيل الحجز** ويجد **نموذج الدفع** ضمن قسم الدفع:
   - اختيار عربون/كامل (إن كانت `depositPercentage > 0`) + طريقة الدفع → "ادفع الآن".
5. عند نجاح الدفع → `confirmed` → ثم `completed` بنفس منطق الشاليه الفوري.
6. إذا رفض المالك (`rejectBooking`) → `cancelled` مع سبب الرفض.

## 4. المهلات التلقائية (Cron)

| المهمة | التكرار | المنطق |
|--------|---------|--------|
| `expirePendingBookings` | كل دقيقة | إلغاء الحجوزات `pending_payment` غير المدفوعة: **الشاليه الفوري بعد 15 دقيقة** (جلسة دفع نشطة)، **الشاليه المؤجل بعد 24 ساعة من الموافقة** (لإعطاء العميل وقتاً للدفع) |
| `autoCompleteBookings` | كل ساعة | تحويل `confirmed` إلى `completed` بعد انتهاء وقت الفترة (يدعم الفترات الليلية) |

> **إصلاح مهم:** سابقاً كان الشاليه المؤجل يُلغى تلقائياً بعد 15 دقيقة من الموافقة (يظهر "مرفوض"
> بدون أي إجراء من العميل)، لأن `updatedAt` يتجدد لحظة الموافقة فيدخل ضمن نافذة الـ15 دقيقة.
> الآن يُميَّز نوع الشاليه فتُعطى الحجوزات المؤجلة مهلة 24 ساعة.

## 5. منطق إظهار/إخفاء العناصر حسب الحالة

### صفحة تفاصيل الحجز للعميل (`booking-success.tsx`)

العناصر تُعرض اعتماداً على `booking.status` و `booking.paymentModel` و `booking.amountPaid`:

| الحالة | الشارة (Badge) | التنبيه | نموذج الدفع | المبالغ المعروضة |
|--------|----------------|---------|-------------|------------------|
| `pending_approval` | "بانتظار موافقة المالك" (برتقالي) | تنبيه: سيراجع المالك طلبك | ❌ مخفي | الإجمالي فقط |
| `pending_payment` | "تمت الموافقة، بانتظار الدفع" (وردي) | — | ✅ ظاهر (عربون/كامل + طريقة الدفع + ادفع الآن) | الإجمالي |
| `confirmed` | "مؤكد" (أخضر) | تنبيه أخضر: مؤكد، تُكتمل بعد الإقامة | ❌ مخفي | المدفوع + المتبقي (إن عربون) |
| `completed` | "مكتمل" (أزرق) | — | ❌ مخفي | المدفوع كاملاً |
| `cancelled` | "ملغي" (أحمر) | تنبيه أحمر: سبب الإلغاء | ❌ مخفي | الإجمالي |

قواعد إضافية:
- **شارة حالة الدفع** تُشتق من الحالة الفعلية (`isPaid = confirmed || completed`) وليس من حقل وهمي.
  إذا كان عربوناً وبقي متبقٍ → "مدفوع جزئياً (عربون)".
- **خيار العربون/الكامل** يظهر فقط عند `depositPercentage > 0`.
- **صف المبلغ المتبقي** يظهر فقط عند: مدفوع + نوع الدفع عربون + المتبقي > 0.
- **معلومات العميل** تُعرض من بيانات الحجز الفعلية (الاسم/الهاتف) لا من قيم ثابتة.

### صفحة تفاصيل الحجز للمالك (`booking-details.tsx`)

| الحالة | زر الإجراء |
|--------|-----------|
| `pending_approval` | "تأكيد الطلب" (موافقة) + "إلغاء الحجز" |
| `pending_payment` / `confirmed` مع متبقٍ > 0 | "تسديد المبلغ المتبقي" (تأكيد استلام نقدي → `completed`) |
| مدفوع بالكامل | "تم سداد المبلغ بالكامل" (معطّل) |
| `cancelled` | بطاقة سبب الإلغاء فقط (لا أزرار) |

- [x] الشاليه المؤجل: الدفع داخل تفاصيل الحجز بعد الموافقة، بنفس طريقة الشاليه الفوري.
- [x] الشاليه المؤجل لا يُلغى تلقائياً بعد الموافقة (مهلة 24 ساعة بدل 15 دقيقة).
- [x] إظهار/إخفاء العناصر (الشارات، التنبيهات، نموذج الدفع، المبالغ) حسب الحالة والدفع.
- [x] الحجز يبقى ظاهر الحالة حتى يُكمله المالك أو تنتهي الإقامة → `completed`.
